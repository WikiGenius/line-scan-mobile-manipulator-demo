# Public / Private Boundary

This document defines what belongs in the public scaffold and what should stay private.

## Public

Safe public content:

- simplified demo code,
- synthetic data,
- high-level explanation,
- toy scripts,
- non-confidential plots,
- candidate metrics,
- public roadmap,
- limitations,
- links to related public repos.

## Private

Keep private:

- full unpublished method,
- paper drafts,
- real lab data,
- advisor notes,
- collaborator notes,
- detailed gap analysis,
- full ablation studies,
- confidential experiment results,
- raw RGB/hyperspectral data,
- calibration details that are part of unpublished experiments.

## Release Rule

Before moving anything from private research work into this public repo, check:

1. Does it reveal an unpublished contribution?
2. Does it include real lab data or calibration details?
3. Does it include advisor/collaborator feedback?
4. Does it copy copyrighted paper figures or tables?
5. Does it make the repo look more mature than it is?

If any answer is yes, keep it private or create a simplified/sanitized public version.
